# LLM採用戦略分析

## 概要

性能/コスト/運用保守の観点から、LLM呼び出し方式を比較検討した結果をまとめる。

---

## 選択肢の比較

### 1. 直接API（OpenAI, Google AI, Anthropic等）

| 観点         | 評価       | 詳細                                                |
| ------------ | ---------- | --------------------------------------------------- |
| **性能**     | ⭐⭐⭐⭐⭐ | 最新モデルへの即時アクセス、レイテンシ最小          |
| **コスト**   | ⭐⭐⭐⭐   | 中間マージンなし、トークン単価のみ                  |
| **運用保守** | ⭐⭐       | 複数プロバイダの管理が必要、API仕様変更への個別対応 |

**メリット:**

- 最新機能への即時アクセス
- プロバイダごとの最適化（キャッシング、バッチ処理等）
- コストの透明性

**デメリット:**

- プロバイダ切り替えにコード変更が必要
- 複数APIキーの管理
- 各社の障害対応を個別に実装

---

### 2. GCP Vertex AI

| 観点         | 評価     | 詳細                                                  |
| ------------ | -------- | ----------------------------------------------------- |
| **性能**     | ⭐⭐⭐⭐ | TPUサポート、バッチ処理に強い（10,000+リクエスト/時） |
| **コスト**   | ⭐⭐⭐   | 直接APIより若干高め、隠れたインフラコストあり         |
| **運用保守** | ⭐⭐⭐⭐ | GCPエコシステム統合、MLOps機能充実                    |

**価格例（2026年2月）:**

- Gemini 3 Flash GA: $0.30/1M入力、$2.50/1M出力
- Gemini 2.5 Pro: $1.25/1M入力、$10.00/1M出力

**メリット:**

- Geminiモデルに最適化
- AutoML機能（トレーニング時間40-60%削減）
- Google Cloud全体との統合

**デメリット:**

- GCPロックイン
- 他社モデル（GPT、Claude）は利用不可

---

### 3. AWS Bedrock

| 観点         | 評価       | 詳細                                                       |
| ------------ | ---------- | ---------------------------------------------------------- |
| **性能**     | ⭐⭐⭐⭐   | 推論レイテンシ200-800ms、クロスリージョン対応              |
| **コスト**   | ⭐⭐⭐     | 推論ワークロードに25-30%コスト効率、ただし隠れコストに注意 |
| **運用保守** | ⭐⭐⭐⭐⭐ | 統一API、複数プロバイダ対応、AWSサービス統合               |

**価格例（2026年2月）:**

- Llama 3.3 70B: $0.72/1K入出力トークン
- バッチ推論: オンデマンド比50%割引

**メリット:**

- 複数プロバイダ（Anthropic, Meta, Mistral等）を統一APIで利用
- Cross-Region Inferenceによる高可用性

**デメリット:**

- **隠れコスト**: Knowledge Bases（OpenSearch Serverless）で月$600-700の固定費
- AWSロックイン

---

### 4. Vercel AI Gateway + SDK

| 観点         | 評価       | 詳細                                                    |
| ------------ | ---------- | ------------------------------------------------------- |
| **性能**     | ⭐⭐⭐⭐   | ルーティングレイテンシ<20ms、ストリーミング最適化       |
| **コスト**   | ⭐⭐⭐⭐   | モデル利用料は直接API同等（BYOK）、インフラコストに注意 |
| **運用保守** | ⭐⭐⭐⭐⭐ | Next.js統合、開発者体験最高、ゼロ運用                   |

**価格構造:**

- AI Gateway自体: 追加料金なし（BYOK）
- Proプラン: $20/月/ユーザー

**メリット:**

- Next.js + Vercelとのシームレス統合
- ストリーミングUIの実装容易性
- 複数LLMプロバイダの抽象化

**デメリット:**

- サーバーレス関数タイムアウト（最大5分）

---

## 推奨構成

### **Vercel AI SDK + 直接API（BYOK）**

| 理由               | 詳細                                         |
| ------------------ | -------------------------------------------- |
| 技術スタック整合性 | Next.js + Vercel + Firebase と最も相性が良い |
| 開発効率           | `useChat`, `streamText`等で実装工数削減      |
| コスト効率         | BYOKにより直接APIと同等コスト                |
| 柔軟性             | 統一APIで複数プロバイダ切り替え可能          |
| 運用負荷           | フルマネージド、インフラ管理不要             |

### なぜ Vertex AI / Bedrock を選ばないか

| サービス    | 課題                                   |
| ----------- | -------------------------------------- |
| Vertex AI   | GPT/Claudeが使えない、GCPロックイン    |
| AWS Bedrock | 既存インフラと不整合、隠れコストリスク |

| ---------- | ---------------------- | -------------------- |

| 統合層     | Gemini 3 Pro / GPT-5.2 | Google/OpenAI直接API |
