# Issue 006: ログイン情報のキャッシュによる無限ローディングの発生と修正

## 概要
ローカル環境にて `npm run dev` 実行後、`localhost` にアクセスすると、画面中央にローディングインジケーター（ぐるぐる表示）が表示されたまま、アプリケーションが進行しなくなる不具合が発生しました。

## 原因の特定
この問題は、ローカル環境に保存された過去のログインキャッシュ（GoogleやFirebaseの認証情報）と、新しく追加されたFirestoreの利用制限機能（`users` コレクションの `status: active/pending` チェック）の間のデータの不整合、そしてその際のエラーハンドリング不足が原因でした。

1. キャッシュによって `onAuthStateChanged` が発火し、`firebaseUser` が存在すると判定される。
2. その `email` を用いてFirestoreの `getUserData` を呼び出す。
3. 未登録データベース、またはルール上読み取りが拒否されるなどのエラー（あるいはその他の通信エラー）が発生した際、非同期処理を実行する関数本体がクラッシュ。
4. 関数の終端に記述されていた `setLoading(false)` が実行されず、ReactのContextが `loading: true` の状態でロックされる。
5. `<AuthGuard>` 等の依存コンポーネントがローディング画面を表示し続けてしまう。

## 修正内容
`src/lib/auth-context.tsx` に以下の変更を実施しました。
*   `onAuthStateChanged` のコールバック関数内で実行される非同期処理全体を `try-catch-finally` ブロックでラップするように修正。
*   エラー発生時 (`catch` ブロック) は、コンソールにエラーを出力しつつ、ユーザーのステートを `null`（未ログイン状態）に安全にリセットする。
*   `finally` ブロック内で確実に `setLoading(false)` を呼び出すことで、いかなる場合でもローディング状態を解除し、未ログインの場合は正しいルート（`/login` 画面）へと戻るように保証した。
