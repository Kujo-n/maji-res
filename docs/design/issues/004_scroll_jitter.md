# 概要
回答生成中（LLMからのストリーミング受信中）に、チャット画面が小刻みに上下にガクガクと震える現象（ジッター）が発生しています。
ユーザーがメッセージの読み込み中に画面の揺れにより文章が読みづらくなるため、ユーザー体験を著しく損ねている状態です。

# 発生条件と再現手順
1. 何らかのプロンプトを入力し、MAGIからの回答パースを開始する
2. 回答のストリーミング中、画面の最下部付近にスクロールが自動で追従する際に、画面がガクガクと震える

# 原因
`src/components/chat/chat-view.tsx` にて、メッセージの更新ごとに以下のような `useEffect` がトリガーされています。

```tsx
  useEffect(() => {
    messagesEndRef.current?.scrollIntoView({ behavior: "smooth" });
  }, [messages, agentResponses, verdict]);
```

`behavior: "smooth"`（滑らかなスクロールアニメーション）が指定されているため、ストリーミングによって高頻度で `messages` 等のステートが更新されるたびに、前のアニメーションが完了する前に新しいアニメーションの命令が割り込みます。
この割り込み処理の連続によって、スクロール位置の計算が衝突もしくはリセットされ、画面が小刻みに上下に揺れる現象（ジッター）が発生しています。

# 提案される解決策
以下のいずれかのアプローチで修正を検討します。

### 案A: `behavior: "smooth"` を外す（または更新中だけオフにする）
シンプルに `{ behavior: "auto" }` に変更することで即座に下に張り付くようになり、アニメーションの競合によるガタつきは解消されます。

### 案B: スマート・スクロール（追従型）の実装（推奨）
「ユーザーが現在ページの一番下付近を見ている場合のみ自動で下へスクロールし、ユーザーが過去のメッセージを読むために上へスクロールしている間は下に強制移動させない」という制御を入れます。最近のチャットUI（ChatGPTなど）の標準的な実装です。
ストリーミング中はアニメーションをオフにするか、あるいは更新頻度を間引く（スロットリング）などの工夫もこれに組み込みます。

# 関連ファイル
- `src/components/chat/chat-view.tsx`

# 対応方針と結論
**案B（スマート・スクロールの実装）** を採用し、問題を解決しました。

### 実装の詳細
1. **自動スクロールの制御（スマートスクロール）**
   - ユーザーが画面を上にスクロールして過去のメッセージを読んでいる時は、自動スクロール（追従）を停止する制御（`isAutoScrollEnabled` ステート）を導入しました。
   - 新規メッセージを送信したタイミングで再び自動スクロールがオンになるように実装しています。
2. **スクロールアニメーションの変更**
   - ストリーミング時は更新頻度が非常に高く、`behavior: "smooth"`（アニメーション）では前後の動作が衝突してガタつきの原因となるため、追従オンの時は `behavior: "auto"`（即時移動）に変更しました。
   - これにより、ストリーミング中の不要なアニメーション計算が排除され、ジッター（ガタつき）が完全に解消されました。

以上をもって本件をクローズとしました。
