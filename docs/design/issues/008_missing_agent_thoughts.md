# 事象調査レポート：エージェントの回答・思考内容が表示されない不具合について

## 概要
チャット送信時にユーザー入力内容は表示されるものの、3エージェントの回答内容および思考内容が画面上に一切レンダリングされない不具合が発生しました。本件は複数の独立したバグとAPI制限が重なって発生した複合的なインシデントです。

## 発生事象と原因

本不具合は以下の4つの要因が複合して発生していました。

### 1. レンダリング状態更新でのRace Condition (UIストリーミングバグ)
- **原因**: `use-magi-stream.ts` にて、ストリーミングデータを受信するたびに配列の最後尾（`length - 1`）を上書きする実装になっていました。非同期処理中に新規スレッドロードや他の状態変更が挟まると、最後尾が対象のアシスタントメッセージと一致しなくなり、意図しないメッセージが上書き・消去される状態不整合が発生していました。
- **対応済**: `updateMessages` において配列の最後尾を指定するのではなく、メッセージの `id` をキーにして `map` で確実に一致するメッセージのみを更新するように修正しました。

### 2. UIのハードコーディングによる表示制限 (UIレンダリングバグ)
- **原因**: `LayeredStack.tsx` コンポーネントにおいて、描画対象の役割が `MELCHIOR`, `BALTHASAR`, `CASPER` の3つにハードコーディングされていました。そのため、「侍」「ロボット」「お気持ち」など新しいカスタムプリセットの動的な役割名を受け取った際、データは存在していてもUI側でフィルタリングされてしまい、非表示となっていました。
- **対応済**: 役割名が何であっても、受信した `responses` 配列をそのままマップして描画し、役割のインデックスに応じてテーマカラーを動的に割り当てるように修正しました。

### 3. Firestoreへの保存エラー (バックエンドデータバグ)
- **原因**: 新しいプリセットや特定の状況下で、エージェントからのレスポンスに `vote` や `needsClarification` のメタデータが含まれていない（またはパースに失敗した）場合、`undefined` が発生していました。Firestoreの `addDoc` は `undefined` を許容しないため保存処理がクラッシュし、状態更新プロセスが中断されていました。
- **対応済**: `decision-service.ts` において、`undefined` になる可能性のあるフィールドに `?? null` （Null合体演算子）を追加し、Firestoreの保存エラーを防止しました。

### 4. Gemini APIの無料枠クォータ超過 (外部API制限)
- **原因**: 上記のバグを修正したのちも回答が生成されない現象が続きました。Next.jsサーバーのログを解析した結果、`gemini-2.5-flash` モデルの **無料枠APIリクエスト上限（`generate_content_free_tier_requests`, limit: 20）** に到達し、Google API側からエラー `429 Too Many Requests (RESOURCE_EXHAUSTED)` が返却されてストリーミングが強制終了されていました。
- **対応（ユーザー側作業待ち）**: この制限はソースコード側では回避不能です。APIの利用可能回数がリセットされるまで待機するか、有料プラン設定済みのAPIキーに変更する必要があります。

## 現在のステータス
ソースコード上のバグ（原因1〜3）はすべて特定・修正済みです。API利用制限（原因4）の解除または切り替え後に、改めて動作確認を実施する予定です。
