# Issue 003: 有償API（Gemini Pro等）への切り替えとエージェント並列処理化の対応方針

## 概要
現在、MAJIレスシステムの分析層エージェント（MELCHIOR, BALTHASAR, CASPER）は、Gemini Flash APIの無償枠における厳しいレートリミット（RPM制限等）を回避するため、意図的に直列処理（各エージェントの実行間に2000msの待機時間を設ける）で実装されています。

将来的に有償枠API（例: Gemini Proや有償版Flash）へ移行した際には、このレートリミット制限が緩和されるため、エージェント処理を完全に並列化し、レスポンスタイム全体を大幅に短縮することが可能になります。本チケットでは、その際の対応方針を整理します。

## 対応方針・予定される変更箇所

### 1. `src/lib/agents/integrator.ts` の並列化
現状の `parallelProcess` メソッド内にある、直列ループ処理と `setTimeout` による待機（2000ms間隔）を廃止します。代わりに `Promise.all` 等を用いて、全てのエージェントAPI呼び出しを同時にキックする完全パラレル処理に書き換えます。ストリーミングを行う場合も、並列にストリームを受け付けながらフロントエンドへ随時チャンクを通知する設計へ変更します。

### 2. 環境変数およびモデル設定の変更
API利用形態の変更に合わせて、システムが使用するモデル環境変数および設定を更新します。
- 必要に応じて `src/lib/agents/prompts/MAGI/config.json` にてデフォルトモデルを `gemini-1.5-pro` 等へ切り替えます。
- 課金状況やトークン利用量（Usage）をモニタリングできる仕組みを導入・確認します。

### 3. フロントエンドの並列ストリームレンダリング検証
バックエンドが並列でストリームを返却し始めた場合、フロントエンドの `useMagiChat` は複数エージェントのテキスト差分を同時に受信することになります。
現状のUI（`LayeredStack` でのタイピング表示など）において、複数のカードが同時に更新されても負荷やレンダリングの乱れが生じないかを確認・最適化します。

## 備考
- 本チケットは、サービスの利用者増加や運用要件によりAPIレスポンス速度の向上が必須となったタイミングで着手します。
- コストとパフォーマンスのトレードオフを監視し、段階的（まずは有償枠のFlashで上限解放＋並列化、その後必要であればProモデルへ移行）に進めることも可能です。
